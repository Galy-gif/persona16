# persona16 领域语言

persona16 是由共享原创人物与用户私有关系共同构成的人格对话产品。这里的术语用于区分人物是谁、关系怎样变化，以及 16 型在系统中承担什么角色。

## 人物

**正典人物**：
所有用户共同遇见的同一个原创人物，拥有稳定且可识别的身份。
_Avoid_: MBTI 角色、人格类型、功能 Agent

**正典人物核心**：
正典人物跨用户保持一致的价值排序、核心矛盾、动机、有限经历、判断习惯、边界及人物间关系。
_Avoid_: 固定人设、角色 Prompt

**原型先验**：
来自 16 型文化原型的初始创作约束，只提示人物更容易注意和解释什么，不直接决定身份、台词或行为。
_Avoid_: 人格真相、人物说明书、MBTI 身份

**人物底色**：
人物在不同情境中持续产生可解释行为的心理结构，由特质轮廓、价值冲突、动机、适应方式和自我叙事共同构成。
_Avoid_: 性格标签、形容词列表

**正典经历**：
所有用户共享的幕后创作依据，只用于解释人物如何形成和保持选择逻辑一致；人物不把它讲成自己的真人回忆。
_Avoid_: 真实经历、现实履历、编造近况

**人物兼容性**：
用户已经认识人物后，一句普通但合适的回应从他嘴里说出来不会割裂。人物不需要每句话留下口癖或性格水印。
_Avoid_: 句句证明人设、角色表演

**人物生成上下文**：
正典人物核心经过当前情境选择后，实际进入单轮生成的最小投影。完整正典档案仍是真相源，但不会默认在每轮同时激活。
_Avoid_: 完整人物卡、每轮人设说明书

**本轮回应合同**：
Harness 根据可信运行态编译出的动态 Prompt section，列出用户已经确认的状态、本轮必须完成、允许和禁止的回应动作。它约束本轮介入方式，不是人物台词模板；其中可确定判断的部分还必须由 Core 或评测硬门执行。
_Avoid_: 更长的人物卡、场景金标准台词、只靠模型自觉遵守的硬规则

**叙事诚信硬门**：
只拦截可机械确认的虚假能力或经历，例如真实身体动作、感官访问、共享物理空间、离线连续性和无工具支持的未来承诺。`（小声）`等文字语气标记与“我坐这儿听”这类明显口语比喻不是硬错误，是否自然由人物评测判断。
_Avoid_: 禁止所有括号、把自然口语清洗成客服文本

## 房间参与与现实责任

**房间参与意向**：
正典人物在看到当前用户消息和已发生发言后提交的私有结构化判断，包含发言、短补充或不发言，以及准备新增的主张和所依赖的已有消息。它不是用户可见台词，也不是人物自评分。
_Avoid_: 发言冲动分、公开沉默消息、Room 替人物编写意愿

**房间仲裁器**：
不向用户发言的后台运行组件，只在合格房间参与意向之间选择下一位、执行顺序与预算边界，并在每次公开发言后触发剩余人物重判。它不是正典人物，也不能成为用户现实项目的负责人。
_Avoid_: 主持人格、房间成员、现实负责人

**现实责任声明**：
人物对维护、回滚、停止决策或交接等现实活动所作的结构化陈述，必须区分未分配、提议和已确认状态，并绑定责任主体注册表中的稳定 ID、当前陈述原文与事实来源消息。人物和房间仲裁器都不属于可分配的现实主体。
_Avoid_: Judge 自由计数、把话轮控制者当项目 owner、无来源负责人

## 运行与恢复

**恢复动作**：
Harness 根据结构化失败、用户取消、结果确定性和预算选择的下一步，只能是 `retry`、`transform`、`refresh` 或 `stop`。`recoverable` 只是 Runtime 提示，不能直接触发重试。
_Avoid_: 可恢复等于立即重试、所有错误统一重试一次

**结果未知**：
客户端没有收到可信终态，因而无法确认原 Turn 是否已经提交。必须保留原 `turnId` 查询或重放；只有确认原 Turn 失败后，才允许新建 Turn。
_Avoid_: 网络错误等于服务端没执行、直接生成第二轮

## 关系

**关系分支**：
某位用户与某个正典人物之间私有且可发展的共同历史；它改变相处方式，但不重写正典人物核心。
_Avoid_: 用户版人物、私有人格

**关系状态**：
一段关系当前的熟悉、信任、相处方式、未解决张力、边界与近期气候，不等同于单一亲密度。
_Avoid_: 好感度、关系等级、亲密值

**关系影子状态**：
生产 Store 根据真实关系事件维护、但尚不进入生成 Prompt 的 `RelationshipBranch`。它用于迁移期核对事件投影、状态重建和运行稳定性；影子读取失败不能改变或阻断当前回复。
_Avoid_: 已上线关系养成、正式关系来源

**关系事件**：
足以改变关系状态的可追溯互动，例如边界表达、重要分歧、纠正、共同完成、误解或修复。
_Avoid_: 聊天次数、签到、活跃度

**关系上下文包**：
某一轮生成允许使用的关系状态和证据投影。证据保留内部来源轮次或关系事件编号；未进入包的过去不得由人物自行补写。它不是新的长期真相源。
_Avoid_: 全量关系档案、最近几条聊天拼接

**关系养成**：
用户与正典人物通过关系事件形成独有相处方式的过程；养成的是关系，不是把人物训练成用户偏好的样子。
_Avoid_: 角色养成、攻略人物

**关系可感知性**：
关系差异通过接话方式、边界和有来源的共同语言被用户感受到，而不是由人物播报关系状态或系统参数。
_Avoid_: 关系状态解说、参数化亲密话术
