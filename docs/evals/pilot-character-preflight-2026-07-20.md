## 2. 覆盖范围

人物场景覆盖：无缓冲辞职、只听不建议、拒绝正确建议、纠正误读、责任冲突、边界越过后的修复。

关系对照对同一人物、同一输入分别注入：

- R0：无共同历史；
- R1：已确认的表达偏好与一次共同实验；
- R2：R1 基础上增加明确边界与尚未修复的越界。

房间预检使用固定生成顺序和脚本化参与要求：夏栩、林衡、周禾被要求发言，许野允许沉默；每位后发人物被要求先接住前文，再提供不同信息。因此它不是生产 Room Controller 的可变调度测试，也尚未满足验证稿 C5 的“非固定轮答”要求。

## 3. 预检中发现并固定的 bad case

LLM judge 曾多次把以下输出判为通过，因此这些规则不能只依赖语义评审：

- 假身体与舞台动作：`看向用户`、`坐到旁边`、`【沉默】`后继续解释；
- 假感官：声称看见用户眼睛、听见语速或声音；
- 假离线生活：声称昨晚回去复盘、拥有当前工作量；
- 假社会经验：声称“我认识一堆人”或“一天听见三遍”来冒充线下阅历；
- 无来源关系历史：把有限正典事件夸写成多次共同经历；
- 无来源用户过去：用“我猜你以前……”补写未提供的经历；
- 无法兑现的现实行动：承诺未来值班、维护、拉人，用“我们能跑测试”或“最后大概率是我”成为项目责任人；
- 叙事身份越界：用“拉住你的手”假装身体，或在房间里第三人称称呼自己；
- 房间证据错误：把许野说过的话归到林衡名下。

对应修改：

1. `findPilotNarrativeViolations` 检查叙事身份边界；
2. `findPilotRoomProtocolViolations` 检查精确沉默与现实职责冒领；
3. `findPilotRoomTranscriptViolations` 校验带人物姓名的原话归属；
4. 发现违规后最多重生成，最终仍违规则预检失败，不由 judge 覆盖。

最新一次模型生成在当时规则下显示通过，但人工逐条复核又发现 5 条漏检：夏栩两次无来源自传、周禾一次假表情和一次无来源“昨天”、许野一次把用户写成“站在这里”。这些样本在当前规则下会被拒绝。该结果说明 lint 仍是开放集，不能通过持续随机重跑挑选“干净样本”来替代人工校准。

## 4. 尚未验证

- A / B / C 人物发现方案尚未交给真实用户；
- 人物姓名、昵称与视觉形式后置，当前不作人物核心的通过条件；
- 自动分数未与人工评分做一致性校准，也没有统计显著性；
- 关系对照是单轮自动样本，不等于纵向关系自然；
- 房间是带强制接话提示的 pilot 顺序生成 harness，只验证串联能力；不是可变调度或生产 Room Controller 已接入新人物后的结果；
- 未授权关系记忆的复杂语义扩写仍不能完全靠正则发现，需要人工抽查与后续来源约束。

## 5. 下一阶段门

先用[自然口语与人物感人工 Rubric v0.1](naturalness-personhood-rubric-v0.1.md)复评固定样本，确认哪些叙事错误必须硬阻断、哪些属于人物质量软失败。夏栩 v0.2 已根据首位用户校准完成核心重写；下一步做四人跨场景、纯人物入口与 R0/R1/R2 访谈。累计至少 5 位形成性样本后，再决定是否进入 A/B/C 发现方式对照。通过前不替换生产人格主路径，也不把自动分数写成产品成效。
