# 正典人物与私有关系分支

persona16 的前台身份采用共享原创人物，16 型只作为内部原型先验；每位用户与同一正典人物拥有私有关系分支。这个选择在类型辨识的低门槛、人物长期连续性和用户关系个体化之间取平衡：人物核心不会被用户训练成另一人，关系却可以因可追溯事件靠近、疏远、冲突或修复；关系不使用单一好感度或聊天次数驱动。

## Consequences

- L1 必须从“选择一种类型”改为“发现一个人物”。
- 人物评测以人物辨识、合理变化和关系特异性为核心，MBTI 辨识降为内部护栏。
- 现有单轴 `intimacy` 关系模型需要由多维、事件驱动的关系状态替代。
- 正典人物核心、人物间关系和用户私有关系分支必须分开存储和版本化。
