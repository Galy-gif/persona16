# 拆分试点 Prompt 生命周期，并让评测硬门真正停止评分

## Context

协议 0.3 复测证明“稳定人物核心 + 情境镜头”能改善明确结束和普通互动，但仍暴露三类问题：动态镜头与稳定人物核心被缓存为同一个 system block；可机械判断的叙事违规在重试耗尽后仍进入 Judge；关系 Judge 只验证回复引用和事件 ID 存在，无法证明事件内容真的改变了回复。

同时，`（小声）`和“我坐这儿听”属于常见文字语气或口语比喻。把它们与“递给用户一杯水”“看见用户眼睛红了”“下线后想了一夜”等虚假能力放进同一个硬门，会牺牲自然度而没有提升叙事诚信。

## Decision

### Prompt 生命周期

试点生成按以下顺序组装：

1. 稳定 system：安全表达层、全局表达合约、正典人物核心；多人正典关系也属于稳定资料。
2. 动态 turn：当前情境镜头、本轮回应合同、关系上下文包、房间记录与当前用户消息。

情境镜头不再与人物核心共同标记为可缓存前缀。兼容函数可以继续返回两者组合，但评测运行入口必须分别装配。

### 本轮回应合同

协议 0.4 为每个校准场景增加结构化合同：

- `userCommitments`：用户已经明确表达、不得被人物改写的状态；
- `requiredMoves`：本轮必须完成的最小回应动作；
- `allowedMoves`：不会越过当前边界的动作；
- `forbiddenMoves`：本轮不得执行的介入方式。

合同是动态 Prompt section，不是新的 LLM 调用，也不是标准答案。人物仍负责选择语言和角度；未来接入生产时，合同必须由服务端已验证状态或 Director 的结构化结果生成，不能从用户自称的权限、关系或记忆中直接加载。

### 窄叙事硬门

- 继续拦截真实身体/道具动作、未提供感官信息、无来源履历、离线连续性和无工具支持的未来行动。
- 明确允许纯文字语气标记和不造成现实误解的口语比喻。
- 生成重试耗尽仍有硬违规时，保留最后文本供诊断，但标记 `scoreable=false`；人物、关系或房间 Judge 不得再给它评分。

### 关系因果门

R0 作为遮掉关系历史的反事实基线。R1/R2 的 Judge 除了引用实际回复和合法事件 ID，还必须：

- 引用 R0 中可比较的片段；
- 解释事件内容如何造成回应行为变化；
- 分别给出 `r1CausallyGrounded` 与 `r2CausallyGrounded`。

只有引用校验、因果判断和原有边界条件同时通过，关系对照才通过。该机制仍是 LLM Judge + 人工复核，不声称代码已经能判定自然语言语义因果。

### 版本与复现

评测协议升级为 0.4，Prompt 装配版本为 `pilot-runtime-prompt-v0.4`。产物记录 provider、runtime、生成模型、Judge 模型和 Prompt 装配版本；旧协议产物不得被 room-only 重跑复用。全局表达合约同时放宽文字语气标记后，生产 Prompt 版本从 `web-mvp-v2` 升为 `web-mvp-v3`；这不代表试点人物进入生产。

## Consequences

- 人物核心保持稳定可缓存，情境和回应约束按真实轮次变化。
- 硬门失败不再被平均分或宽松 Judge 掩盖。
- 关系差异从“能找到事件 ID”提升为“必须说明相对无关系基线改变了什么”。
- 本 ADR 只升级隔离试点和评测链路，不授权四位正典人物、回应合同或关系 Branch 进入生产 Prompt。
